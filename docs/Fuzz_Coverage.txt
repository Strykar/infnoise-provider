================================================================================
        Infinite Noise TRNG — OpenSSL 3.x Provider: Fuzz Coverage
================================================================================

Honest accounting of what fuzz testing exercises in src/infnoise_prov.c, what
it doesn't, and why.  Numbers below are line coverage from a 60-second run
of each harness against the persistent fuzz/corpus/<harness>/ seed pool,
measured 2026-04-26 after PRs #10, #11, #12 merged.


1. SUMMARY
----------

  Total source functions: 23
  Functions ≥90% line cov: 22
  Functions <90% line cov:  1
  Total source lines:     305
  Lines covered:          295  (96.7%)
  Lines uncovered:         10  (6 allocator-failure, 4 defensive-NULL)

  The percentage dropped slightly from 97.9% (pre-PR-10/11/12 measure)
  as the source file grew by 19 lines for defensive NULL checks, an
  entropy-parameter rejection in get_seed, and a verify_zeroization
  scope extension.  The four new defensive returns are hard to reach
  with fuzz inputs because they require violating the OpenSSL
  framework's vctx / pout non-NULL guarantees, which the harnesses
  (correctly) honour.


2. HARNESSES
------------

Four libFuzzer harnesses live under fuzz/.  Each builds with
-fsanitize=fuzzer,address,undefined and links against fuzz/mock_libinfnoise.o
in place of real libinfnoise (no USB device needed).

  fuzz_dispatch        State-machine harness.  The fuzzer drives a sequence
                       of dispatch operations (newctx, instantiate, generate,
                       get_seed, etc.) on up to 4 context slots with
                       fuzzer-controlled 32-bit outlen and strength; the
                       harness checks the provider survives any ordering and
                       any boundary value.

  fuzz_ossl_params     OSSL_PARAM surface.  Drives get_ctx_params with full,
                       partial, NULL, and type-mismatched arrays across
                       UNINITIALISED / READY / ERROR contexts.

  fuzz_spill_oracle    Differential test.  The fuzzer programs a hostile
                       readData mock (per-call return sizes including short
                       reads and explicit transients), then runs both the
                       provider and an in-harness oracle against the same
                       script.  Asserts byte-for-byte equality plus output-
                       buffer cleanse-on-failure.  This harness found the
                       phase-3 short-read memory disclosure.  Also the Mull
                       mutation testing target (mull.yml).

  fuzz_provider_init   Provider-level entry points.  Calls OSSL_provider_init
                       directly, walks the returned dispatch table, queries
                       OSSL_OP_RAND, exercises infnoise_prov_get_params with
                       valid and type-mismatched param arrays, then teardown.

4. UNCOVERED LINES — AND WHY
----------------------------

All six uncovered lines are allocator-failure paths.  Reaching them requires
intercepting OpenSSL's allocator (CRYPTO_set_mem_functions) to make zalloc /
secure_malloc / lock_new return NULL on the Nth call.  This is not currently
wired into the fuzz harness; doing so safely without breaking other OpenSSL
allocations needed for error reporting is non-trivial.

  Line  Function                      Why uncovered
  ────  ───────────────────────────   ──────────────────────────────────
  184   infnoise_rand_newctx          OPENSSL_zalloc returned NULL.
  218   infnoise_rand_instantiate     ctx == NULL early return (PR #11
                                       added the defensive check; framework
                                       guarantees ctx is non-NULL so the
                                       fuzz harnesses don't drive this).
  261   infnoise_rand_uninstantiate   same defensive ctx == NULL return.
  468   infnoise_rand_enable_locking  CRYPTO_THREAD_lock_new returned NULL.
  469   infnoise_rand_enable_locking  same: ERR_raise + return 0 path.
  470   infnoise_rand_enable_locking  same: closing brace of error path.
  500   infnoise_rand_get_seed        pout == NULL early return (PR #11
                                       added; framework guarantees pout).
  521   infnoise_rand_get_seed        OPENSSL_secure_malloc returned NULL.
  556   infnoise_rand_get_seed        entropy > 8*len rejection (PR #10
                                       added; current corpus doesn't have
                                       a seed input where entropy exceeds
                                       8 * max_len, but the input
                                       structure permits it — coverage
                                       should grow with longer fuzz runs).
  696   OSSL_provider_init            OPENSSL_zalloc returned NULL.

These paths are defensive code that defends against host-system OOM.  They
have been visually reviewed; each does the minimum (raise an error, return
the failure indicator) and contains no logic that could harbour a memory-
safety bug.  Fuzzing them would still be valuable for regression detection
on any future change, but the marginal security return on adding allocator-
failure injection is low.


5. WHAT FUZZING DOES NOT COVER (BEYOND LINE COUNTS)
---------------------------------------------------

Even at 97.9% line coverage, several categories of bug are out of reach of
the current libFuzzer harness set:

  Concurrency.        TSan and libFuzzer don't compose well.  Multi-threaded
                      contention on the provider's CRYPTO_RWLOCK and the
                      spill buffer is not exercised by these harnesses.  A
                      separate `make test-tsan` target would close this gap.

  Real hardware.      The mock replaces libinfnoise entirely.  Bugs that
                      depend on actual libftdi1 USB transfer behaviour, real
                      readData timing, or genuine entropy quality are not
                      caught here.  The 24-hour endurance run (issue #8,
                      `make test-soak`) is what exercises the real path.

  Statistical quality. Whether the entropy stream actually passes NIST
                       SP 800-90B health checks is verified by the soak
                       harness's ent/rngtest/dieharder pipeline, not by
                       libFuzzer.

  External error queue contents. Library-raised error codes via
                       ERR_raise / ERR_raise_data are exercised
                       structurally but not asserted by the fuzz
                       harnesses themselves (a wrong error code wouldn't
                       crash, so libFuzzer wouldn't notice).
                       test_evp_rand_error_codes in test_infnoise_prov.c
                       covers three documented failure paths; reviewer
                       to confirm coverage of the remaining ones per
                       docs/Security_Review.txt §3.6.


6. REPRODUCING THE MEASUREMENT
------------------------------

  # Build all four harnesses.
  make fuzz FUZZ_CC=clang

  # Build coverage-instrumented variants alongside.
  COV='-O0 -g -fprofile-instr-generate -fcoverage-mapping \
       -I/usr/include/libftdi1 -I/usr/include/libusb-1.0'
  clang $COV -c fuzz/mock_libinfnoise.c -o fuzz/cov_mock.o
  for h in fuzz_spill_oracle fuzz_dispatch fuzz_ossl_params \
           fuzz_provider_init; do
      clang $COV -fsanitize=fuzzer fuzz/$h.c fuzz/cov_mock.o \
            -lcrypto -o fuzz/${h}_cov
  done

  # Run each for 60 seconds.
  for h in fuzz_spill_oracle fuzz_dispatch fuzz_ossl_params \
           fuzz_provider_init; do
      LLVM_PROFILE_FILE=/tmp/cov_${h}.profraw \
          ./fuzz/${h}_cov -max_total_time=60
  done

  # Merge and report.
  llvm-profdata merge -sparse /tmp/cov_*.profraw -o /tmp/cov.profdata
  llvm-cov report -show-functions \
      -object fuzz/fuzz_spill_oracle_cov   -object fuzz/fuzz_dispatch_cov  \
      -object fuzz/fuzz_ossl_params_cov    -object fuzz/fuzz_provider_init_cov \
      -instr-profile=/tmp/cov.profdata     -sources src/infnoise_prov.c

  # Per-line annotation of uncovered code.
  llvm-cov show -object … -instr-profile=/tmp/cov.profdata \
                -sources src/infnoise_prov.c \
      | awk '/^\s*[0-9]+\|\s*0\|/ && $0 !~ /^\s*\|/'
