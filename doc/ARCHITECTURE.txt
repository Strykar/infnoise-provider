================================================================================
        Infinite Noise TRNG — OpenSSL 3.x Provider: Architecture
================================================================================

1. OVERVIEW
-----------

This provider registers an OSSL_OP_RAND seed source backed by the Infinite
Noise TRNG over USB.  When configured as the DRBG seed source in
openssl.cnf, every OpenSSL cryptographic operation (key generation,
signatures, TLS handshakes) is seeded with hardware entropy.

  Provider:    src/infnoise_prov.c    (OSSL_OP_RAND implementation)
  Tests:       test/test_infnoise_prov.c    (27 tests, 5 layers)
  Config:      conf/infnoise-provider.cnf   (loads provider + sets seed)

A legacy ENGINE-API implementation by Tim Skipper exists at
tinskip/infnoise-openssl (dormant since 2020).  This provider is a
ground-up rewrite for the Provider API that replaced the ENGINE API in
OpenSSL 3.x; it shares no code with the legacy implementation.


2. PROVIDER STRUCTURE
---------------------

The provider implements the OSSL_OP_RAND operation with the following
dispatch entries:

  newctx / freectx                Per-context lifecycle
  instantiate / uninstantiate     Open/close the USB device
  generate                        Produce entropy into caller buffer
  reseed                          No-op (TRNG cannot be reseeded)
  enable_locking / lock / unlock  CRYPTO_RWLOCK serialization
  gettable_ctx_params / get_ctx_params
  get_seed / clear_seed           Secure-allocated seed buffers for DRBGs
  verify_zeroization              Constant-time check post-uninstantiate

Per-context state is held in a heap-allocated PROV_INFNOISE struct
(OPENSSL_zalloc on newctx, OPENSSL_clear_free on freectx):

  struct PROV_INFNOISE {
      void                    *provctx;
      struct infnoise_context  trng_context;   // libinfnoise state
      SpillBuffer              spill;          // 64-byte batch carry
      int                      state;          // UNINIT/READY/ERROR
      CRYPTO_RWLOCK           *lock;
  };


3. SPILL BUFFER AND THREE-PHASE GENERATE
----------------------------------------

libinfnoise's readData() produces exactly BATCH_SIZE (64) bytes per call
when configured with multiplier=2 and Keccak whitening.  Caller requests
can be any size from 1 byte to INFNOISE_MAX_REQUEST (1 MiB).  The spill
buffer bridges this mismatch.

generate() proceeds in three phases:

  Phase 1 — drain spill
    If the spill buffer holds leftover bytes from a previous call,
    copy them into the caller's output first.  Cleanse them from the
    spill buffer as they are consumed.

  Phase 2 — direct batch copy
    While the remaining output length is >= BATCH_SIZE, have readData
    write directly into the output buffer.  Zero intermediate copies,
    zero spill buffer usage.

  Phase 3 — tail with stash
    If output length is not a multiple of BATCH_SIZE, read one final
    batch into a stack buffer, copy the needed tail bytes to the
    caller's output, and stash the remainder in the spill buffer for
    the next call.  Cleanse the stack buffer.

This is simpler than a ring buffer (flat array, no wrap-around) and
avoids a memcpy for large requests that are multiples of BATCH_SIZE.


4. THREAD SAFETY
----------------

libinfnoise (the patched fork required by this provider) holds Keccak
and health-check state per-context, so multi-context concurrent use is
limited only by FTDI USB exclusion — one open handle per physical
device.  A single PROV_INFNOISE shared between threads is serialised
via CRYPTO_RWLOCK (allocated on enable_locking); the caller takes
lock()/unlock() around dispatch entries that touch shared state.

The TSan target (make test-tsan) stresses both topologies: 4 threads
sharing one context with locking, and 4 threads each owning a context.


5. SECURITY INVARIANTS
----------------------

  No stale entropy in memory
    OPENSSL_cleanse on spill buffer after drain, on stack buffers after
    use, on output buffer on any error, on the entire PROV_INFNOISE
    struct at freectx.

  No partial output on error
    generate() cleanses the entire output buffer before returning 0 on
    any error path.  Callers never see partial TRNG data mixed with
    uninitialized memory.

  Bounded request size
    max_request is capped at 1 MiB (INFNOISE_MAX_REQUEST).  Advertised
    via get_ctx_params and enforced in generate().

  Bounded retry on transient failures
    readData returning 0 bytes (transient) is retried up to 100 times,
    then the context transitions to ERROR.

  Resource cleanup on all paths
    freectx deinits the TRNG on both READY and ERROR states.
    OPENSSL_clear_free zeroes the context before release.

  Secure seed allocation
    get_seed allocates via OPENSSL_secure_malloc; clear_seed uses
    OPENSSL_secure_clear_free.

    Caveat — mlock is the application's responsibility.  OpenSSL's
    secure heap is opt-in: it must be initialised by the application
    via CRYPTO_secure_malloc_init() before any provider call.  When
    the secure heap is uninitialised, OPENSSL_secure_malloc silently
    falls back to plain malloc — no mlock, no swap protection, no
    additional protection beyond a regular allocation.  Applications
    that want mlock'd seed buffers must initialise the secure heap
    themselves (typical: 65536-byte arena before any EVP_RAND use).
    The provider cannot do this on the application's behalf because
    secure heap setup must precede ALL OpenSSL allocations process-
    wide; calling it from inside a provider would be too late.

  Constant-time zeroization check
    verify_zeroization uses CRYPTO_memcmp against a zero buffer.  The
    compiler cannot optimise this away after OPENSSL_cleanse, unlike a
    memcmp-then-loop.

  Strength enforcement
    instantiate and generate reject requests exceeding the advertised
    256-bit security strength.


6. LIBINFNOISE REQUIREMENTS
---------------------------

Upstream libinfnoise (waywardgeek/infnoise) holds its Keccak sponge and
health-checker state in file-scope static variables, and readData()
returns uint32_t with errors signalled via context->errorFlag.  Both
properties prevent safe multi-context use and proper fatal-vs-transient
error discrimination.

This provider hard-requires the patched fork (Strykar/infnoise on the
libinfnoise-error-codes branch as of 2026-04-25), which:

  * Moves Keccak / health-check state into the caller-provided
    struct infnoise_context (per-context isolation).
  * Adds the infnoise_error_t enum.
  * Changes readData() return to int32_t — rc < 0 fatal (the value is
    the infnoise_error_t code), rc == 0 transient (retry), rc > 0 is
    bytes written.

The required marker is INFNOISE_KECCAK_STATE_SIZE.  src/infnoise_prov.c
guards the build:

  #ifndef INFNOISE_KECCAK_STATE_SIZE
  #  error "libinfnoise must be the patched fork ..."
  #endif

The compile-time bridge that previously supported both libraries was
removed; see commit history.  CI (build / sanitizers / cifuzz / codeql /
coverity) clones Strykar/infnoise:libinfnoise-error-codes.


7. BUILD HARDENING
------------------

Flags match the Arch Linux system OpenSSL build:

  CFLAGS:  -fPIC -Wall -Wextra -O2
           -D_FORTIFY_SOURCE=3
           -fstack-protector-strong
           -fcf-protection            (CET/IBT)
           -fstack-clash-protection
           -fno-plt
  LDFLAGS: -shared
           -Wl,-z,relro,-z,now         (Full RELRO)
           -Wl,-z,noexecstack          (NX)
  Final:   strip --strip-unneeded

checksec on the produced infnoise.so reports:
  Full RELRO, Stack Canary, SHSTK & IBT, NX, PIE, stripped, no RPATH.


8. TEST COVERAGE
----------------

27 tests across 5 layers.  All pass under:
  - Normal build (-O2, hardened)
  - AddressSanitizer (detect_leaks=1, abort_on_error=1)
  - UndefinedBehaviorSanitizer (halt_on_error=1)

Layer 1 — Direct libinfnoise hardware (4)
  hw_detect, hw_init_deinit, hw_read_raw, hw_read_multiple

Layer 2a — Provider API, no hardware (6)
  provider_load, provider_params, evp_rand_fetch,
  evp_rand_gettable_params, evp_rand_ctx_uninitialised,
  provider_reload_cycles

Layer 2b — Provider API, hardware required (6)
  evp_rand_lifecycle, evp_rand_ctx_params, evp_rand_locking,
  evp_rand_strength_check, evp_rand_generate_sizes, evp_rand_reseed

Layer 3 — High-level OpenSSL integration (3)
  rand_bytes, rsa_keygen_2048, ec_keygen_p256

Layer 4 — Statistical quality (4)
  stat_monobit (NIST SP 800-22 frequency test, p > 0.01),
  stat_byte_distribution (chi-squared, chi2 < 350),
  stat_runs (serial independence),
  stat_two_sample_independence (two reads differ > 90%)

Layer 5 — Memory safety and robustness (4)
  mem_ctx_churn (100 newctx/freectx cycles),
  mem_instantiate_churn (10 full lifecycle cycles),
  mem_boundary_generate (spill buffer boundary sizes),
  mem_zero_length_generate


9. CONFIGURATION
----------------

conf/infnoise-provider.cnf configures OpenSSL to:
  1. Load the default provider (activate=1)
  2. Load the infnoise provider from the modules directory
  3. Set infnoise as the DRBG seed source: [random_sect] seed=infnoise

Usage:
  export OPENSSL_CONF=infnoise-provider.cnf
  openssl rand 10000000 | ent

Or append the provider/random sections to the system openssl.cnf.
